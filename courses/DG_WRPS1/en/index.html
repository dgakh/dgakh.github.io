About this course

This 2-day course is oriented to the issues relating to software performance, platform effect, algorithms and programming languages as well as actions to improve software products. The effect of software performance on its quality and reliability has also been reviewed.

The course is based on the authorâ€™s 25-year practical experience in developing and optimizing software solutions. The course is the master class on systematization of problems and solutions in the sphere of software performance. The course is intended for programmers and software developers, however, it can be useful for specialist of adjacent spheres such as software architects, managers of IT projects, and QA.

This course is aimed to enable the participants to gain knowledge in the following subjects:

  • Assessment of needs for optimization. Methods of performance evaluation;
  • Features of hardware components and their contribution to the performance of applications;
  • Features of the operating system components and their contribution to the performance of applications;
  • Factors affecting the speed of the algorithms;
  • Features of programming languages and their impact on application performance. Comparing the work of codes implemented on C#, Java, and C++.

The essential part of the course is labs with code exercises demonstrating practical implementation of optimization. The labs are implemented for the following platforms:

  • Programming languages: C#, Java, C++;
  • Operation systems: Windows, Linux;
  • Architectures: .NET Framework, .NET Core, JVM, Native applications;

Course Content

Module 1: Introduction

The issues of assessing the need for optimization of applications, relevance of optimization to quality, factors affecting performance, performance evaluation methods, and advantages and disadvantages of optimization have been covered.

The relation between the processes of optimizing applications with reagard to professional specialization have been covered in this Module.

Module 2: Hardware

Contribution of hardware components, including CPU, motherboard, chipset, memory, I / O devices, hard disk, graphics system, as well as the data structure of hardware and software level with regard to application performance have been concerned in this section.

The examination of this module takes more time compared with the other modules due to the fact that the hardware is a base element and success in optimization depends on the understanding of hardware functioning.

Module 3: Operating Systems

The module reviews the contributions of operating system components to the performance of applications, including the different types of operating systems, the implementation of the processes / threads, files, virtual memory, system calls, etc.

Module 4: Algorithms

This module introduces the contributions of algorithms to the performance. The factors affecting the performance of the algorithms, as well as generalized methods of optimization have been examined, including the contribution of algorithms being the part of other more complex algorithms as well as the affect of sorting algorithms.

Module 5: Programming Languages

Within this module the contributions of programming languages, translation/compilation, runtime environment, language abstraction level to performance have been discussed, including the differences between performance of programs depending on programming languages ( C#, Java, and C++ ).

Module 6: References

Additional sources of information such as books and websites have been proposed for further learning.

Labs

Labs are the essential part of this course. Almost all labs are realized for all consodered platforms. List of labs includes:


Arithmetic Operations. Assessment of primitive arithmetic operations with floating point numbers.

Arithmetic Operations Int. Assessment of primitive arithmetic operations with integral numbers.

Arithmetic Operations Spec. Evaluation of benefits of replacing some arithmetic operations by other operations.

Loop Array. Evaluation of the various cycles.

Loop For vs Foreach. Assessment of various methods of access to collections in different types of cycles.

Vectorization. Acceleration of performance through optimization by vector processor instructions.

Parallelizm. Acceleration of performance through optimization by parallel execution of operations.

Branch Prediction. Evaluation of the CPU's ability to predict branching.

Cache Hit Ratio. Assessing how cache hits impact the code execution.

Cache Actualization.. Assessing how cache actualization impacts the code execution.

Inlining.. Assessing the impact of the inclusion of the function body instead of calling.

MT Locks. Evaluation of different methods of threads locking.

Classes vs Generics. Evaluation of construction and access time for classes and templates.

Classes vs Generics Array. Evaluation of construction and access time for array of classes and templates.

Classes vs Structures. Evaluation of construction and access time for classes and structures.

Exceptions. Evaluation of time of error detection by simple checking and exception mechanism.

Algorithm Precise Reduction. Optimization of algorithm by reducing the accuracy of calculations.

Matrix Mul. Optimization of matrix multiplication algorithm.

Boxing Unboxing. Evaluation of boxing/unboxing cost (C#, Java).

Unsafe Stack. Evaluation of the benefits of location of array in the stack (C#).
